⚡ Complexity-aware routing

Heuristics + a fast free reasoning-model classifier pick a lane per request: FAST, SIMPLE, AGENTIC, HARD, or LONGCTX. Trivial tasks never burn premium-grade models.

🧮 One quota ledger, every machine

A Durable Object tracks each provider's per-minute and per-day budgets. Your laptop and desktop draw from the same ledger — exhausted providers are skipped with zero 429s.

🛟 Survives outages

Ordered fallback chains on 429/5xx/timeout, 10-minute health cooldowns, and mid-stream failover before the first byte. A flaky provider costs you seconds, not your session.

🔐 Your keys, your Worker

Deploys under your Cloudflare account, gated by your bearer token. Keys live as Worker secrets. A privacy guard keeps secret-looking content away from providers that train on inputs.

How it works

Point Claude Code's ANTHROPIC_BASE_URL at your Worker. Every request is parsed (Anthropic wire format, streaming and tool calls included), classified, and forwarded to the best free model whose quota and health allow it — then translated back, so Claude Code never knows the difference.

Lane Serves Default chain (hot-editable YAML)
FAST quick lookups, renames, one-liners gemini-flash-lite → step-3.7-flash → laguna-xs
SIMPLE single-file edits, docstrings laguna-xs → nemotron-49b → mistral-medium
AGENTIC multi-step tool work (default) laguna-s → laguna-xs → glm-5.2 → minimax-m3
HARD deep debugging, architecture deepseek-v4-pro → minimax-m3 → gemini-pro
LONGCTX >60k-token contexts gemini-flash (1M) → nemotron-ultra (1M)

Chains hot-reload from YAML with kompass config push — no redeploy. Paid models are unreachable unless you explicitly set allow_paid: true (enforced in code).

Quickstart

Prereqs: Node 20+, pnpm, a free Cloudflare account, and at least one free provider key (OpenRouter, NVIDIA Build, Google AI Studio, Groq).

1
Clone and install 
git clone https://github.com/vinoth4v/kompass && cd kompass && pnpm install
2
Create a Cloudflare API token (dash.cloudflare.com → My Profile → API Tokens → "Edit Cloudflare Workers" template, plus KV Storage:Edit) 
export CLOUDFLARE_API_TOKEN=your-token-here
3
Run the wizard — it asks for your provider keys, provisions KV + your workers.dev URL, deploys, verifies end-to-end, and installs the claude-free command in your shell: 
pnpm kompass init
4
Code for free 
source ~/.zshrc
claude-free        # Claude Code, now riding free models through your Worker

FAQ

Is this really $0?

Yes for the infrastructure and the models: Cloudflare Workers/KV/Durable Objects free tier comfortably fits a heavy coding day, and every default lane model is a free-tier endpoint. Paid models are blocked in code unless you opt in with allow_paid: true.

Where do my prompts and keys go?

Prompts transit your own Cloudflare Worker (only you hold the bearer token) and go to whichever free provider serves the lane. Keys are Cloudflare Worker secrets under your account. Note: free endpoints may train on inputs — Kompass flags those providers and its privacy guard keeps requests matching your secret/path patterns away from them. Don't point truly sensitive repos at free endpoints.

Does streaming and tool-calling really work?

Yes — the Worker translates the full Anthropic wire protocol both directions (SSE streaming, tool_use/tool_result round-trips, thinking blocks) for both OpenAI-format and Gemini-format providers. It's exercised by 46 tests and a live smoke harness on every release.

What happens when a provider dies mid-session?

Failures before the first byte fall through the chain to the next model; the failing model gets a 10-minute cooldown; your session's context is preserved because Kompass speaks to Claude Code in Anthropic format the whole time.

Does this touch my Claude subscription?

No. Kompass never proxies Anthropic subscription auth (their terms prohibit it). It's a separate command — claude-free — you use when you want to preserve or have exhausted your Pro/Max quota. Your normal claude stays untouched.

Can I self-host without Cloudflare?

The state layer is a pluggable interface; a self-hosted SQLite target is on the roadmap. Today the same codebase also runs locally via wrangler dev.
